Results for PCx using IBM's WSSMP sparse Cholesky package

PCx's version 1.1 allows the user to hook up different sparse system solvers quickly and easily. As an example of how an interface to a solver could be implemented, we now provide code to hook up IBMs WSSMP sparse Cholesky solver. WSSMP was implemented by Gupta, et al. and uses a nested dissection ordering. As a consequence the timing results for some problems are quite different from what we get using the default solver that is provided with PCx. Overall, WSSMP never performs significantly worse, while for some larger problems it outperforms the Ng-Peyton sparse Cholesky solver by as much as an order of magnitude. To illustrate this, we provide some timing results for problems, where the differences are most striking (the runs were performed on Argonne's IBM quad ):
